# Playbooks, Runbooks & Artifacts

A public repository for playbooks, runbooks, and other operational and technical artifacts.

## Overview

This repository serves as a central, versioned store for operational documentation:

| Type | Description |
|---|---|
| **Playbooks** | Strategic guides that describe *what* to do in a given scenario — goals, stakeholders, decision trees, and success criteria. |
| **Runbooks** | Step-by-step procedural instructions for *how* to perform a specific task or respond to an incident. |
| **Other Artifacts** | Supporting documents, diagrams, checklists, templates, and reference material. |

## Directory Structure

```
.
├── playbooks/          # High-level operational playbooks
├── runbooks/           # Step-by-step procedural runbooks
├── templates/          # Reusable templates for new documents
└── CONTRIBUTING.md     # Guidelines for adding or editing artifacts
```

## Getting Started

### Browsing Artifacts

- Browse [`playbooks/`](./playbooks/) for scenario-based guides.
- Browse [`runbooks/`](./runbooks/) for procedural how-to documents.
- Browse [`templates/`](./templates/) to start a new document from a template.

### Creating a New Artifact

1. Pick the right template from [`templates/`](./templates/).
2. Copy it into the appropriate directory (`playbooks/` or `runbooks/`).
3. Fill in every section — leave none blank.
4. Submit a pull request. See [CONTRIBUTING.md](./CONTRIBUTING.md) for details.
